# PostPNG Maker

PostPNG Maker は、PDFをスマホで扱いやすい PNG/JPEG 画像へ変換するブラウザ内完結型PWAです。X投稿向けの4枚セット作成と、汎用的なPDF画像化の通常モードを備えています。

## 現在の重要な制限

この実行環境では npm registry / CDN / GitHub raw への取得がすべてプロキシの `403 Forbidden` で失敗したため、PDF.js、PDF worker、JSZipの公式実体ファイルを自動同梱できていません。完成版として配布する前に、以下の公式配布ファイルを必ず配置してください。

- `libs/pdf.min.js`: `pdfjs-dist@4.10.38/build/pdf.min.mjs` の内容を配置
- `libs/pdf.worker.min.js`: `pdfjs-dist@4.10.38/build/pdf.worker.min.mjs` の内容を配置
- `libs/jszip.min.js`: `jszip@3.10.1/dist/jszip.min.js` の内容を配置

配置後、`libs/` 内に `throw new Error(...)` や未実装メッセージ、外部CDN参照ラッパーが残っていないことを確認してください。実行時に外部CDNへアクセスする構成は完成版として不可です。

## プライバシー

選択したPDFと生成画像は外部サーバーへ送信しない設計です。PDF.js、PDF worker、JSZipを `libs/` にローカル同梱すると、変換処理はブラウザ内だけで実行されます。PDFファイルや生成画像はService Workerのキャッシュ対象にしていません。

## X投稿モードの使い方

1. 「PDFを選択」からPDFを読み込みます。
2. 横幅と余白トリミングを選びます。
3. 必要に応じて投稿本文・ALT下書きを入力します。
4. 「変換する」を押すと、1〜4ページが「X投稿セット1」、5〜8ページが「X投稿セット2」のように4ページごとに分かれます。4ページ未満は1セット、5ページは2セット、8ページは2セット、9ページは3セットになります。
5. 各セットカードで「セットをZIP保存」「本文をコピー」「ALTをまとめてコピー」が使えます。各ページの個別保存とALTコピーもできます。
6. ボトムバーは誤操作を避けるため、変換後は「全保存」「再変換」を中心に表示します。

指定横幅は、画像を縮小するときの上限です。元画像が指定横幅より小さい場合は、画質劣化を避けるため拡大しません。細かい表では2000pxや通常モードの高倍率を検討してください。

ALT下書きは変換結果に紐づきます。PDF変更や再変換を行うと編集内容はリセットされます。

## 通常モードの使い方

通常モードでは、PNG/JPEG、JPEG品質、解像度倍率、ページ範囲、背景色、余白トリミングを選択できます。ページ範囲は `1-4`、`1,3,5`、`2-4,8` のように指定します。不正な範囲やPDFページ数を超える指定はエラー表示します。

JPEGでは透明背景を使用できません。JPEG選択時に透明背景が選ばれていた場合は自動で白背景へ変更され、UIにも注意を表示します。PNGに戻すと透明背景を再び選べます。

## PWA / オフライン利用

初回ロード後、アプリシェルは Service Worker によりキャッシュされ、ホーム画面追加後はPWAとして起動できます。キャッシュ対象はHTML、CSS、JS、ローカルライブラリ、manifest、実在するアイコンのみです。PDFファイルや生成画像、blob URLはキャッシュしません。

オフライン時はUIの起動と、既にキャッシュ済みのアプリ本体の表示が可能です。ただし、初回ロード前やライブラリ未配置の状態では変換できません。

更新が反映されない場合は、ブラウザのサイトデータ/キャッシュ削除、PWAの再追加、または再読み込みを行ってください。Service Workerは install 時に `skipWaiting()`、activate 時に古いキャッシュ削除と `clients.claim()` を行います。

## Android Chromeでの使い方

1. Android Chromeで `index.html` を開きます。
2. メニューから「ホーム画面に追加」または「アプリをインストール」を選択します。
3. ホーム画面のアイコンから起動します。
4. 画面幅360〜430pxの縦画面を想定し、主要ボタンは44px以上、「変換する」は大きめのボタンとして表示されます。

## 大きいPDFの注意

ページ数が多いPDFや高倍率変換はメモリを多く使います。スマホでは20ページ超の一括変換時に確認を表示します。重い場合は倍率を下げる、ページ範囲を分ける、またはPCで実行してください。パスワード付きPDF、破損PDF、対応外形式は読み込みに失敗する場合があります。

## 余白トリミングの制限

白背景では白余白、黒背景では黒余白を検出します。透明背景やカスタム背景では誤検出を避けるため制限があります。トリミング後も最低余白と極端に小さくしない安全策を入れています。

## ファイル構成

- `index.html`: アプリの画面構造
- `css/style.css`: モバイル優先のUIスタイル
- `js/app.js`: 画面制御、モード制御、変換フロー
- `js/pdf-service.js`: PDF.js読み込み、Uint8ArrayでのPDF読み込み、PDFレンダリング
- `js/image-service.js`: リサイズ、Blob生成、余白トリミング
- `js/export-service.js`: ダウンロード、ZIP、コピー処理
- `js/settings-service.js`: 設定保存、structuredCloneフォールバック、ページ範囲解析、日本語ファイル名対応の安全化
- `js/pwa-service.js`: Service Worker登録と更新反映
- `service-worker.js`: PWAアプリシェルキャッシュ
- `manifest.json`: PWAメタデータとアイコン参照
- `icons/`: `icon-192.png`、`icon-512.png`、`maskable-icon-512.png`
- `libs/`: PDF.js、PDF worker、JSZipのローカル配布ファイル配置先

## 使用ライブラリ

- PDF.js / pdfjs-dist: PDFのブラウザ内レンダリング
- JSZip: ZIPファイル生成

## ライセンス

アプリ本体のライセンスはプロジェクトのライセンスに従います。PDF.js と JSZip は各ライブラリのライセンスに従います。配布時は同梱ライブラリのライセンス表記を確認してください。

## 既知の制限

- 現在のリポジトリ状態では、ライブラリ実体ファイルが未同梱のため変換とZIP保存は動作しません。
- パスワード付きPDFは非対応または制限があります。
- PDFによってはフォント、注釈、透明効果の表示が元PDFと完全一致しない場合があります。
- Clipboard APIが使えない環境では、手動コピー用テキストエリアを表示します。
