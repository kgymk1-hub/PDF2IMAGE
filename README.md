# PostPNG Maker

PostPNG Maker は、PDFをスマホで扱いやすい PNG/JPEG 画像へ変換するブラウザ内完結型PWAです。X投稿向けの4枚セット作成と、汎用的なPDF画像化の通常モードを備えています。

## 完成版の状態

PDF.js、PDF worker、JSZip は `libs/` にローカル同梱済みです。外部CDNへ戻さず、初回ロード後はPWAとして利用できます。

- `libs/pdf.min.js`: PDF.js本体
- `libs/pdf.worker.min.js`: PDF.js worker
- `libs/jszip.min.js`: JSZip

## プライバシー

選択したPDFと生成画像は外部サーバーへ送信されず、ブラウザ内で処理されます。PDFファイル、生成画像、blob URL は Service Worker のキャッシュ対象にしていません。

## Android Chromeでの使い方

1. GitHub Pages など、HTTPSで公開したアプリURLを Android Chrome で開きます。
2. ローカルファイルとして `index.html` を直接開く運用は推奨しません。`file://` では ES Modules や Service Worker が動作しない可能性があります。
3. ローカル確認する場合は、プロジェクトルートで簡易HTTPサーバーを起動し、localhost 経由で開きます。

   ```bash
   python -m http.server 8000
   ```

   ```text
   http://localhost:8000/index.html
   ```

4. 「PDFを選択」から端末内のPDFを選びます。
5. 必要なモードと出力設定を選び、「変換する」を押します。
6. 生成された画像を個別保存、セット別ZIP保存、または全ページZIP保存します。

## ホーム画面への追加方法

1. Android Chromeのメニューを開きます。
2. 「ホーム画面に追加」または「アプリをインストール」を選択します。
3. ホーム画面に追加されたアイコンから起動します。
4. 初回ロード後は、キャッシュ済みのアプリシェルによりPWAとして利用できます。

PWA更新が反映されない場合は、ブラウザのサイトデータ/キャッシュ削除、PWAの削除後の再追加、または再読み込みが必要な場合があります。

## トラブルシュート

### ボタンが無反応な場合

設定ボタン、モード切替、PDFファイル選択などが無反応な場合は、まずJavaScriptの初期化エラーを疑ってください。画面上部の起動診断表示、またはブラウザの開発者ツールのConsoleに出ているエラー内容を確認します。

- GitHub Pagesでは、必要に応じて `/index.html` 付きURL（例: `https://kgymk1-hub.github.io/PDF-to-PNG/index.html`）で開き直してください。
- Android Chromeでは、対象サイトのサイトデータを削除してから再読み込みしてください。
- ホーム画面に追加済みのPWAは、古いService Workerやキャッシュを保持する場合があります。PWAを削除してから再追加してください。
- GitHub Pagesはデプロイ後、反映まで少し待ち時間が発生する場合があります。
- Service Workerの古いキャッシュが残ることがあります。ブラウザのサイトデータ削除、強制再読み込み、またはPWAの再追加で解消する場合があります。

## X投稿モードの使い方

1. 「PDFを選択」からPDFを読み込みます。
2. 出力横幅（1200px / 1600px / 2000px / 元サイズ）と余白トリミングを選びます。
3. 必要に応じて投稿本文・ALT下書きを入力します。
4. 「変換する」を押すと、1〜4ページが「X投稿セット1」、5〜8ページが「X投稿セット2」のように4ページごとに分かれます。
5. 各セットカードで「セットをZIP保存」「本文をコピー」「ALTをまとめてコピー」「セット名をコピー」が使えます。
6. 各ページでは個別保存、情報表示、ALT下書き編集、ALTコピーができます。

X投稿モードは、4ページごとにX投稿セットを作り、白背景PNGで出力します。横幅は設定から選択できます。指定横幅は縮小上限であり、元画像が指定横幅より小さい場合は拡大しません。

ALT下書きは変換結果に紐づきます。PDF変更や再変換を行うと編集内容はリセットされる可能性があります。

## 通常モードの使い方

通常モードでは、PNG/JPEG、JPEG品質、解像度倍率、ページ範囲、背景色、余白トリミングを選択できます。

- ページ範囲は `1-4`、`1,3,5`、`2-4,8` のように指定します。
- 空欄または `全ページ` の場合は全ページを変換します。
- 不正な範囲やPDFページ数を超える指定はエラー表示します。
- JPEG品質は60〜92の範囲で指定できます。
- 背景色は白、黒、透明を選べます。ただしJPEGでは透明背景が使えないため、白背景に切り替わります。
- 余白トリミングは簡易自動処理です。背景色やPDF内容によっては期待どおりに切り抜けない場合があります。

## 大きいPDFを扱う場合の注意

ページ数が多いPDFや高倍率変換はメモリを多く使います。スマホでは20ページ超の一括変換時に確認を表示します。重い場合は倍率を下げる、ページ範囲を分ける、またはPCで実行してください。

パスワード付きPDFは非対応または制限があります。パスワード解除後のPDFを選択してください。破損PDFや対応外形式も読み込みに失敗する場合があります。

## 余白トリミングの制限

白背景では白余白、黒背景では黒余白を検出します。透明背景では誤検出を避けるためトリミングを制限しています。トリミング後も最低余白と極端に小さくしない安全策を入れていますが、PDFのデザインによっては手動調整やOFF設定をおすすめします。

## PWA / オフライン利用

PWA / オフライン利用は、HTTPS配信または `localhost` での表示が前提です。`file://` で `index.html` を直接開いた場合、Service Worker は登録できず、オフラインキャッシュも利用できません。

初回ロード後、アプリシェルは Service Worker によりキャッシュされます。キャッシュ対象はHTML、CSS、JS、ローカルライブラリ、manifest、実在するアイコンのみです。

Service Worker はインストール時に新しいキャッシュを作成し、有効化時に古いキャッシュを削除します。`manifest.json` は `icon-192.png`、`icon-512.png`、`maskable-icon-512.png` を参照します。

## ファイル構成

- `index.html`: アプリの画面構造
- `css/style.css`: モバイル優先のUIスタイル
- `js/app.js`: 画面制御、モード制御、変換フロー
- `js/pdf-service.js`: PDF.js読み込み、Uint8ArrayでのPDF読み込み、PDFレンダリング
- `js/image-service.js`: リサイズ、Blob生成、余白トリミング
- `js/export-service.js`: ダウンロード、ZIP、コピー処理、コピー失敗時フォールバック
- `js/settings-service.js`: 設定保存、structuredCloneフォールバック、ページ範囲解析、日本語ファイル名対応の安全化
- `js/pwa-service.js`: Service Worker登録と更新反映
- `service-worker.js`: PWAアプリシェルキャッシュ
- `manifest.json`: PWAメタデータとアイコン参照
- `icons/`: `icon-192.png`、`icon-512.png`、`maskable-icon-512.png`、`icon.svg`
- `libs/`: PDF.js、PDF worker、JSZipのローカル同梱ファイル

## 使用ライブラリとライセンス

- PDF.js / pdfjs-dist: PDFのブラウザ内レンダリング。ライセンスはPDF.js / pdfjs-distの配布条件に従います。
- JSZip: ZIPファイル生成。ライセンスはJSZipの配布条件に従います。

アプリ本体のライセンスはプロジェクトのライセンスに従います。同梱ライブラリの著作権表示とライセンス表記は、各公式配布元の条件を確認してください。

## 既知の制限

- パスワード付きPDFは非対応または制限があります。
- PDFによってはフォント、注釈、透明効果の表示が元PDFと完全一致しない場合があります。
- JPEGでは透明背景が使えません。
- Clipboard APIが使えない環境では、手動コピー用テキストエリアを表示します。
